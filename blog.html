Frequently Asked Questions

Flexbox: The Flexible Box Layout Module, makes it easier to design flexible responsive layout structure without using float or positioning.
Key Points: (i) One Dimensional (ii) Focuses on content flow content first (iii) Width(or height) of flex items are determined by the content of the item (iv) Spacing and aligning items are flexible

Grid-Layout: The CSS Grid Layout Module offers a grid-based layout system, with rows and columns, making it easier to design web pages without having to use floats and positioning.
Key Points: (i) Multi Dimensional (ii) Focuses on content placement layout first (iii) Provides a mechanism to divide available viewport for layout into columns and rows (iv) Each item can be positioned anywhere in a grid cell

Bootstrap: Bootstrap comes with a set of pre-styled responsive, mobile-first components that possess a definite UI kit.
Key Points: (i) Sites created using Bootstrap follow the generic pattern that makes them look identical. (ii) Websites created with Bootstraps are known for their responsiveness and flawless design, but the looks are generic and similar. (iii) Bootstrap is efficient and saves a lot of time. It has been around for more than nine years and being the most popular CSS Framework, it has a large community of developers, forums, tools, and so on. (iv) Spotify, Twitter, and Lyft are some of the famous companies that use Bootstrap.

Tailwind: Tailwind CSS uses a set of utility classes to create a neat UI with more flexibility and uniqueness.
Key Points: (i) Tailwind offers predesigned widgets to build a site from scratch with fast UI development. (ii) Websites created using Tailwind CSS are much more customizable. (iii) Tailwind CSS is relatively newer, and there is still much room to grow in terms of its community, however, it is growing day by day, and the number of users, references, tools, and websites related to it are also increasing. (iv) Tailwind CSS is used by BlaBlaCar, Exyplis, and Bazzite.

Event Loop: We know,JavaScript is a single Threaded Language and being single Threaded it's work synchronous way. But after being single Threaded it's also works asynchronous way with the help of Browser and Event Loop.

Event Loop Work: Event loop is a mechanism that allows JavaScript to perform non-blocking operations. When an asynchronous task is started, the event loop will start running. Once the task is completed, the event loop will again check for any other tasks that need to be performed.

if it's asynchronous then it's not console. next synchronous data will be console. with the help of Browser API like (DOM, AJax etc) asynchronous data will be work and browser send them callback queue and the event loop check stack is empty or not. if all synchronous work done and stack is empty then with the help of event loop from callback queue to stack data will transfar and then the data will console.

8 Ways to get undefined: We can get undefined from the 8 reasons below:

1) Variable that is not initialized will give undefined.
2) Function with no return will give undefined.
3) Parameter that is not passed will be undefined.
4) If return has nothing on the right side then will return undefined.
5) Property that doesn't exists on an object will give result undefined.
6) Accessing array elements outside of the index range will give result undefined.
7) Deleting an element inside an array will give undefined.
8) Set a value directly to undefined.
